1. 현재 상태

Global Patient Hub는 한국 의료관광 외국인 환자 유치 플랫폼의 MVP입니다. 단순 병원 디렉토리가 아니라 International Patient Marketplace + Coordinator CRM + Quote/Booking Engine 방향으로 설계되어 있습니다.

Public Site 홈, 병원/시술 탐색, 비교, 상담 신청, EN/JP 피부 패키지 랜딩, 플로팅 상담 버튼, Top 버튼.
Internal Admin 베타 운영, 케이스 대시보드, 견적/보증금/예약 MVP, 랜딩 라우트 관리.
Data 정적 seed data + Supabase lead/case 저장 구조 + local demo fallback.
Deployment GitHub main push 후 Vercel production 자동 배포.
Access note: /admin/*, /partner/*, and /provider/* now pass through the internal operations gate. The gate validates the entered role token against the server-side operations API before rendering internal screens. Keep confidential medical documents out of the frontend until full user-account authentication and private storage policies are added.

4. 수정 매뉴얼

메뉴 수정

client/src/components/Navbar.tsx의 navLinks를 수정합니다. 새 문구가 필요하면 I18nContext.tsx에 번역 키를 추가합니다.

홈 수정

client/src/pages/Home.tsx와 client/src/contexts/I18nContext.tsx의 publicHome.* 키를 수정합니다. 내부 운영 정보는 공개 홈에 넣지 않습니다.

번역/언어 수정

I18nContext.tsx에서 LANGUAGES와 T를 수정합니다. Arabic은 dir: "rtl", 나머지는 ltr로 유지합니다.

플로팅 연락 버튼 수정

공식 연락 채널은 VITE_CONTACT_WHATSAPP_URL, VITE_CONTACT_LINE_URL, VITE_CONTACT_WECHAT_URL, VITE_CONTACT_KAKAO_URL 환경변수로 교체합니다. 코드에는 개발용 fallback만 유지합니다.

피부 패키지 SKU 수정

client/src/lib/wedgeData.ts의 SKIN_PACKAGE_SKUS를 수정합니다. 랜딩에 노출하려면 SKIN_LANDING_PAGES의 packageIds에도 연결합니다.

랜딩 페이지 추가

client/src/lib/wedgeData.ts의 SKIN_LANDING_PAGES에 locale/slug row를 추가하면 /{locale}/{slug} 라우트가 생성됩니다.

병원/시술 데이터 수정

client/src/lib/sampleData.ts에서 병원, 시술, 언어, specialty label을 수정합니다.

상담 폼 수정

Consultation.tsx에서 UI와 form state를 수정하고, 저장 필드는 client/src/lib/supabase.ts의 InquiryInput과 insert mapping도 같이 수정합니다.

Supabase schema 수정

기존 migration은 수정하지 말고 supabase/migrations/20260610_0001_...처럼 새 migration을 추가합니다.

Closed beta/admin 데이터 수정

client/src/lib/betaData.ts를 수정합니다. 해당 데이터는 아직 seed/demo 운영 데이터입니다.

6. 운영 가드레일

  • 치료효과를 보장하는 문구를 쓰지 않습니다.
  • "최고", "1위", "무조건 안전" 같은 무근거 비교/우월 표현을 피합니다.
  • 검증되지 않은 병원을 verified로 표시하지 않습니다.
  • 의료비와 비의료비는 분리합니다.
  • 내부 라우트, 베타 지표, Supabase/RLS 정보는 공개 홈에 노출하지 않습니다.
  • 민감한 의료문서/여권정보는 private storage와 접근권한 적용 전 수집하지 않습니다.
